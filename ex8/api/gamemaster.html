GameMaster documentation¶

class gameMaster.GameMaster¶

This holds all of the state in the game, including methods to help accessing and modifying members (such as torpedos, asteroids etc.)

You need to instantiate it only onec throughout - This was already done for you, look at the asteroidsMain.py file

To control the game you can use your keyboard controllers like so:

“Left”, “Right”,”Up” - Movement

” ” -(space key) Fire a torpedo

“q” - Quit the game

__init__()¶

Initializes all of the containers needed for this class to work.

add_asteroid(x, y, xSpeed, ySpeed, size)¶

This adds an asteroid into the game

Parameters:
  • x (int) – The x coordinate of the asteroid.
  • y (int) – The y coordinate of the asteroid.
  • xSpeed (int) – The current speed on the x direction.
  • ySpeed (int) – The current speed on the y direction.
  • size (int) – The size of our asteroid, this is for default asteroids. The available sizes are 3(big), 2(medium) and 1(small).

This method is used to create a repeating action in your game.

set_initial_ship_cords(xCor, yCor, xSpeed=0, ySpeed=0)¶

Sets the initial coordinations for our ship, allows to set a starting speed for both x and y directions.

  • ySpeed (int) – (OPTIONAL) Sets the starting speed in the y direction.
